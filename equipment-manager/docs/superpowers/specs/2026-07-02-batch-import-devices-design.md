# Batch Import Devices via Excel/CSV

**Date:** 2026-07-02
**Status:** Approved

## Overview

Add a batch import feature to EquipHub that lets users upload an Excel (.xlsx) or CSV file to create multiple devices at once. A downloadable template file is provided to guide data entry. Before committing, a preview dialog shows which rows are valid and which have errors.

---

## Architecture

All parsing and DB logic lives in the **Electron main process** (same pattern as catalog thumbnail handling). The renderer only orchestrates the multi-step dialog and passes the file path over IPC — it never receives raw binary file data.

Three new IPC channels, all behind `auth_guard` with the existing `edit_device` permission:

| Channel | Description |
|---|---|
| `devices.downloadTemplate` | Generate .xlsx template and open `showSaveDialog` |
| `devices.previewImport` | Parse + validate a file path; return per-row preview data |
| `devices.importBatch` | Insert pre-validated rows in a single SQLite transaction |

---

## Excel Template

Generated by `xlsx` (SheetJS) on the main process. Columns (in order):

| Column | Header | Required | Notes |
|---|---|---|---|
| A | SKU | Yes | Unique identifier |
| B | Tên thiết bị | Yes | Device name |
| C | Loại | No | Category name (must match an existing category exactly) |
| D | Nhóm | No | Group name within the category (must match) |
| E | Số serial | No | Serial number |
| F | Ghi chú | No | Free-text notes |

Template includes a header row (bold) and one sample data row (greyed) to show expected format.

## Preview Import Flow

1. User clicks **"Chọn file"** → Electron `dialog.openFile` with filters for `.xlsx`, `.xls`, `.csv`
2. File path sent to `devices.previewImport`
3. Main process:
   - Parses the file with `xlsx.readFile()`
   - Resolves category names → IDs and group names → IDs from live DB data
   - Validates each row:
     - SKU non-empty
     - Name non-empty
     - SKU not already in DB
     - SKU not duplicated within the file itself
     - If category name given: must exist
     - If group name given: must exist and belong to the resolved category
   - Returns `{ rows: PreviewRow[] }` where each row has all fields plus `valid: boolean` and `error?: string`
4. Renderer shows the **preview dialog** (Step 2):
   - Table listing all rows; valid rows have a green checkmark, error rows have a red X and inline error message
   - Summary line: "X dòng hợp lệ · Y dòng lỗi"
   - "Nhập X thiết bị" button (disabled if X = 0), "Huỷ" button
5. On confirm → `devices.importBatch` with only the valid rows
6. Main process inserts all rows in a **single SQLite transaction** (all-or-nothing for the valid set)
7. Renderer shows success toast or error banner, invalidates `devices` query cache

## IPC Types (additions to `electron/shared/ipc.ts`)

```ts
// devices.downloadTemplate
interface DownloadTemplateArgs {}        // no args
interface DownloadTemplateResult { saved: boolean }  // false = user cancelled dialog

// devices.previewImport
interface PreviewImportArgs { filePath: string }
interface PreviewRow {
  rowNum: number
  sku: string
  name: string
  category: string           // as typed in file
  group: string              // as typed in file
  categoryId: number | null  // resolved; null if blank or not found
  groupId: number | null
  serialNumber: string
  notes: string
  valid: boolean
  error: string | null       // human-readable Vietnamese error description
}
interface PreviewImportResult { rows: PreviewRow[] }

// devices.importBatch
interface ImportBatchArgs {
  rows: Array<{
    sku: string
    name: string
    categoryId: number | null
    groupId: number | null
    serialNumber: string | null
    notes: string | null
  }>
}
interface ImportBatchResult { imported: number }
```

## Error Handling

- If the file cannot be parsed (unsupported format, corrupt file): return a single top-level error, not per-row errors
- Per-row errors are in Vietnamese, e.g.:
  - `"SKU không được để trống"`
  - `"Tên thiết bị không được để trống"`
  - `"SKU đã tồn tại trong hệ thống"`
  - `"SKU bị trùng lặp trong file (dòng X)"`
  - `"Loại thiết bị không tồn tại: [tên]"`
  - `"Nhóm không thuộc loại đã chọn: [tên]"`
- If `importBatch` transaction fails: return error; no partial inserts
